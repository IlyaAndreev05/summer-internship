# Инструкция администратора — ALINA GPSS AI Консультант

## 1. Требования к окружению

Для запуска системы необходимы:

| Компонент | Минимум |
|---|---|
| Docker | 20.10+ |
| Docker Compose | 2.0+ |
| Оперативная память | 12 ГБ (8 ГБ — Ollama, 1 ГБ — Qdrant, 512 МБ — PostgreSQL, 2 ГБ — приложение) |
| Свободное место на диске | 10 ГБ (модели Ollama ~3 ГБ, данные БД, документация) |
| ОС | Linux, macOS, Windows (через WSL2) |

> **Примечание:** на macOS и Windows Docker Desktop необходимо выделить достаточно памяти в настройках (Preferences → Resources → Memory ≥ 12 ГБ).

## 2. Установка и настройка

### 2.1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd summer-internship
```

### 2.2. Создание конфигурации

```bash
cp .env.example .env
```

Откройте `.env` в текстовом редакторе и настройте переменные (см. таблицу ниже).

### 2.3. Запуск контейнеров

```bash
docker compose up -d
```

При первом запуске Ollama автоматически скачает модели LLM и эмбеддингов. Это занимает 1–3 минуты. Чтобы следить за прогрессом:

```bash
docker compose logs -f ollama
```

Дождитесь сообщений о загрузке `qwen2.5:1.5b` и `nomic-embed-text`, затем нажмите `Ctrl+C`.

### 2.4. Проверка статуса

```bash
docker compose ps
```

Все четыре сервиса должны быть в состоянии `healthy`:

| Сервис | Ожидаемое состояние |
|---|---|
| `app` | healthy |
| `ollama` | healthy |
| `qdrant` | healthy |
| `postgres` | healthy |

### Переменные окружения

| Переменная | По умолчанию | Описание |
|---|---|---|
| `APP_MODE` | `console` | Режим запуска: `console` — интерактивный чат, `vk` — VK бот, `batch` — пакетная обработка вопросов из файлов |
| `LLM_PROVIDER` | `ollama` | Провайдер LLM (поддерживается только `ollama`) |
| `LLM_MODEL` | `qwen2.5:1.5b` | Название модели Ollama для генерации ответов |
| `LLM_BASE_URL` | `http://ollama:11434` | URL Ollama-сервера. Внутри Docker используйте имя контейнера `ollama`; для локального запуска — `http://localhost:11434` |
| `EMBED_MODEL` | `nomic-embed-text` | Модель для генерации векторных эмбеддингов |
| `QDRANT_URL` | `http://qdrant:6333` | URL сервера Qdrant |
| `QDRANT_COLLECTION` | `alina_docs` | Имя коллекции в Qdrant для хранения векторов |
| `POSTGRES_URL` | `postgresql://alina:alina@postgres:5432/alina` | Строка подключения к PostgreSQL |
| `VK_TOKEN` | — | Токен VK-сообщества (обязателен только при `APP_MODE=vk`) |
| `VK_GROUP_ID` | — | ID VK-группы (обязателен только при `APP_MODE=vk`) |
| `DOCS_DIR` | `docs` | Путь к директории с документацией |
| `PROJECTS_DIR` | `projects` | Путь к директории с файлами проектов |
| `TESTS_DIR` | `tests` | Путь к директории с тестовыми файлами |
| `CHUNK_SIZE` | `500` | Размер текстового фрагмента (чанка) в символах при индексации |
| `CHUNK_OVERLAP` | `100` | Перекрытие соседних чанков в символах |
| `CHAT_MAX_MESSAGES` | `20` | Максимальное количество сообщений в истории диалога |
| `CHAT_VERBOSE` | `false` | Показывать детали поиска при ответе (`true`/`false`) |

## 3. Подключение источников данных

Система поддерживает два типа источников:

### 3.1. Документация (`docs/`)

Файлы в форматах: `.txt`, `.md`, `.pdf`, `.docx`, `.doc`.

Поместите руководства пользователя, справочные материалы и документацию по GPSS в директорию `docs/`.

### 3.2. Проекты (`projects/`)

Файлы в форматах: `.xlsx`, `.xls`, `.csv`.

Поместите файлы проектов, таблицы и данные в директорию `projects/`.

### 3.3. Автоматическая индексация

Индексация документов происходит **автоматически** при каждом запуске контейнера `app`. Ручная команда индексации не требуется.

Принцип работы:

1. При старте контейнера `docker-entrypoint.sh` запускает авто-индексацию.
2. Система сканирует директории `docs/` и `projects/` на наличие поддерживаемых файлов.
3. Для каждого файла вычисляется SHA-256 хеш (путь + содержимое).
4. Хеши сравниваются с записями в PostgreSQL (таблица `indexed_files`).
5. **Новые файлы** — загружаются, разбиваются на чанки, векторизуются и сохраняются в Qdrant + PostgreSQL.
6. **Изменённые файлы** — старые векторы удаляются из Qdrant, файл переиндексируется.
7. **Удалённые файлы** — векторы и чанки удаляются из обоих хранилищ.
8. **Без изменений** — файл пропускается (сообщение «All N files up to date»).

## 4. Подключение нового репозитория Git с обновлёнными данными

Если документация ALINA GPSS обновилась и находится в отдельном Git-репозитории:

```bash
# Клонировать или обновить репозиторий с документацией
git clone <url-репозитория-документации> /tmp/alina-docs

# Скопировать файлы в директорию проекта
cp /tmp/alina-docs/*.txt docs/
cp /tmp/alina-docs/*.md docs/
cp /tmp/alina-docs/*.pdf docs/

# Перезапустить контейнер приложения — авто-индексация подхватит изменения
docker compose restart app
```

Проверить результат индексации:

```bash
docker compose logs app | tail -20
```

В логах вы увидите количество проиндексированных файлов и чанков.

## 5. Запуск приложения

### 5.1. Запуск всех сервисов

```bash
docker compose up -d
```

### 5.2. Ожидание готовности Ollama

```bash
docker compose logs -f ollama
```

Дождитесь, пока обе модели будут загружены (`qwen2.5:1.5b` и `nomic-embed-text`).

### 5.3. Интерактивная консоль

```bash
docker compose exec -it app uv run alina-rag console
```

Внутри консоли доступны команды:

| Команда | Действие |
|---|---|
| `/exit` | Выход из консоли |
| `/clear` | Очистить историю диалога |
| `/verbose` | Включить/выключить отображение результатов поиска |

### 5.4. Пакетная обработка (batch)

Поместите файлы с вопросами в `tests/`. Формат: CSV или Excel с колонками `№`, `Вопрос`, `Ответ` (колонка «Ответ» заполняется автоматически).

```bash
docker compose exec app uv run alina-rag batch
```

Результат сохраняется в файлы `*_filled.*` в директории `tests/`.

### 5.5. VK бот

Для запуска VK-бота укажите в `.env`:

```bash
VK_TOKEN=ваш_токен_сообщества
VK_GROUP_ID=123456789
APP_MODE=vk
```

Запуск:

```bash
docker compose exec app uv run alina-rag vk
```

## 6. Типичные ошибки и решения

### Ollama не готова / модели не загружены

**Симптом:** приложение падает с ошибкой подключения к Ollama.

**Решение:**
```bash
docker compose logs ollama
```
Убедитесь, что модели загружены. Если нет — подождите или перезапустите:
```bash
docker compose restart ollama
```

### Недостаточно памяти (Out of Memory)

**Симптом:** контейнеры падают, в логах `OOMKilled`.

**Решение:** увеличьте лимиты памяти в `docker-compose.yml` или освободите ресурсы. Рекомендуется минимум 12 ГБ RAM.

### VK-бот не отвечает

**Симптом:** бот запущен, но не реагирует на сообщения.

**Решение:**
1. Проверьте, что `VK_TOKEN` и `VK_GROUP_ID` указаны корректно в `.env`.
2. Убедитесь, что в настройках VK-сообщества включены «Сообщения» и настроен Callback API.
3. Проверьте логи: `docker compose logs app`.

### Система отвечает «информация не найдена»

**Симптом:** на вопросы по GPSS система не находит ответы.

**Решение:**
1. Убедитесь, что в `docs/` есть файлы с документацией.
2. Проверьте логи авто-индексации: `docker compose logs app | grep -i index`.
3. Перезапустите контейнер для переиндексации: `docker compose restart app`.

### Медленный ответ

**Симптом:** ответ генерируется более 30 секунд.

**Решение:** модель `qwen2.5:1.5b` оптимизирована для скорости, но на слабом оборудовании генерация может занимать время. Проверьте загрузку CPU. При необходимости уменьшите `CHUNK_SIZE` для сокращения контекста.
