# Инструкция пользователя — ALINA GPSS AI Консультант

## 1. Как запустить приложение

### 1.1. Запуск через Docker

Если система уже установлена администратором, подключитесь к контейнеру и запустите интерактивную консоль:

```bash
docker compose exec -it app uv run alina-rag console
```

После запуска вы увидите приветствие:

```
╔════════════════════════════════════════════╗
║    ALINA GPSS AI Консультант               ║
╚════════════════════════════════════════════╝
/exit — выход    /clear — очистить историю    /verbose — показать поиск
```

### 1.2. Команды внутри чата

| Команда | Что делает |
|---|---|
| `/exit` | Завершить сеанс |
| `/clear` | Очистить историю текущего диалога (система забывает предыдущие вопросы) |
| `/verbose` | Включить или выключить режим отладки — показывает, сколько источников найдено при каждом поиске |

## 2. Как задать вопрос

Задавайте вопросы на **русском языке**, связанные с системой GPSS и средой ALINA GPSS.

### Примеры вопросов

**Справочные вопросы** (определения и понятия):
- «Что такое транзакт в GPSS?»
- «Что такое одноканальное устройство (FACILITY)?»
- «Чем отличается FACILITY от STORAGE?»

**Практические вопросы** (как что-то сделать):
- «Как создать транзакт в GPSS?»
- «Как задать время обслуживания?»
- «Как занять устройство в GPSS?»

**Вопросы по примерам** (конкретные случаи):
- «Как работает блок TRANSFER?»
- «Как использовать SAVEVALUE?»
- «Пример модели с очередью и обслуживающим устройством»

### Советы по формулировке

- Используйте точные термины GPSS (GENERATE, SEIZE, ADVANCE и т.д.) — это повышает точность поиска.
- Если ответ не устраивает, попробуйте переформулировать вопрос другими словами.
- Система помнит несколько последних сообщений — можно уточнять: «А как насчёт многоканальных устройств?»

## 3. Как интерпретировать ответ

### 3.1. Структура ответа

Система ищет релевантные фрагменты в документации и генерирует ответ на основе найденной информации. Ответ содержит:

- **Прямой ответ** на ваш вопрос.
- **Источники** — система ссылается на документацию, из которой взята информация.

### 3.2. Если информация не найдена

Если система не нашла ответ в документации, она сообщит об этом. Возможные причины:

- Вопрос выходит за рамки GPSS (система специализируется только на GPSS).
- В документации действительно нет информации по этому вопросу.
- Вопрос сформулирован нестандартно — попробуйте использовать другие слова.

### 3.3. Режим отладки (/verbose)

При включённом режиме отладки система показывает техническую информацию о поиске:

```
🔍 Найдено: 3 (вектор) + 2 (BM25)
```

Это означает, что семантический поиск нашёл 3 релевантных фрагмента, а поиск по ключевым словам — 2. Эта информация полезна, если ответ кажется неполным.

## 4. Известные ограничения

- **Размер модели:** система использует модель `qwen2.5:1.5b` (1.5 млрд параметров). Это компактная модель, оптимизированная для скорости. Ответы могут быть менее подробными, чем у больших моделей.
- **Тематика:** система знает только о GPSS и ALINA GPSS. Вопросы по другим темам (программирование на Python, математика и т.д.) останутся без ответа.
- **Документация:** качество ответов зависит от полноты загруженной документации. Если нужной информации нет в `docs/` — система не сможет её найти.
- **Время ответа:** генерация ответа занимает от 2 до 30 секунд в зависимости от сложности вопроса и нагрузки на сервер.
- **Контекст диалога:** система помнит только несколько последних сообщений. Длинные цепочки рассуждений могут «забываться».
